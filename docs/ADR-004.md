# ADR-004: github-catalog — Repository Intelligence (Extraction & Classification)

**Status:** Proposed
**Date:** 2026-06-18
**Environment:** WSL2 · Bash 5.0+ · jq 1.7 · git · gh (optional enrichment)
**Builds on:** [ADR-001](ADR-001-github-catalog-rewrite.md), [ADR-002](ADR-002.md)

---

## Context & goal

Today a `repo_snapshot` captures identity, HEAD, key files, and verbatim README semantics. The goal of this ADR is to **enrich every snapshot with deterministic, verifiable classification** — *what kind of project is this?* — so that an owner's repos can be filtered and understood at a glance: archetype booleans (AI-related, web app, desktop app, console app, MCP server, bootstrapping-AI app, …), primary language, and health/activity signals.

Hard constraints carried from ADR-001/002:

- **Deterministic, no LLM in the core.** Same input → same labels, every run. Evidence must be inspectable.
- **Zero new core dependency.** Classification runs in the datafetcher we already have, over the bare clone we already fetch. No SQLite, no ML runtime, no Python.
- **Minimal interface.** Classification rides on `sync`; the report gains *one* compact column, not a new command.
- **Append-only & additive schema.** New optional `classification` object; old records still validate.

---

## The question this ADR answers: *can `gh` classify, or do I need another agent?*

**`gh` / the GitHub API cannot classify project archetypes.** GitHub exposes only:

- `language` — the single top language, computed by **Linguist** from file extensions (no notion of "web app" vs "CLI" vs "MCP server").
- `topics` — *manually* applied tags; usually empty or inconsistent across owners.
- `isArchived`, `isFork`, `isTemplate`, `licenseInfo`, `description` — useful metadata, but not archetype.

So the booleans you want (AI-related, MCP, desktop app, bootstrapping-AI app, …) are **not** retrievable from `gh`. They must be **derived locally**.

**You do not need an LLM or a separate agent for the deterministic facets.** Archetype is almost entirely decidable from **manifest and signature files** — `package.json` dependencies, `pyproject.toml`/`requirements.txt`, `Cargo.toml`, `go.mod`, `Dockerfile`, `*.tf`, `mcp.json`, `*.ipynb`, etc. — all of which already sit in the bare clone the datafetcher makes. A rule/signature classifier (a lookup table + jq) covers them with full determinism and verifiable evidence.

An LLM is only worth involving for **fuzzy, subjective labels** that signatures can't decide (e.g. "is this a serious production service or a weekend experiment?", free-text capability summaries). That is explicitly kept **out of the zero-dependency core** and offered as an *optional, clearly-marked, non-deterministic enrichment* behind the MCP boundary in [ADR-005](ADR-005.md) — never mixed into the deterministic `classification` object.

## Competitor scan (what to borrow, what to avoid)

| Tool | What it does | Borrow | Avoid |
|------|--------------|--------|-------|
| **SOMEF** — *Software Metadata Extraction Framework* ([repo](https://github.com/KnowledgeCaptureAndDiscovery/somef)) | Extracts metadata from README + package files using regex, header analysis, **and supervised ML classifiers** | Its taxonomy of metadata fields; manifest-aware extraction across many languages | ML classifiers (non-deterministic, heavy deps) — we use signatures instead |
| **GitHub Linguist** ([repo](https://github.com/github-linguist/linguist)) / **enry** ([repo](https://github.com/src-d/enry)) | Language detection from extensions + heuristics; **enry needs no full Git checkout** | Extension→language mapping; vendored/generated skip lists; enry's "no full clone" model fits our bare-clone tree exactly | Ruby/Go runtime — reimplement a *small* extension histogram in jq, not all of Linguist |
| **github-to-sqlite** (Dogsheep) ([repo](https://github.com/dogsheep/github-to-sqlite)) | Pulls repos/stars/issues/commits/workflows into **SQLite + Datasette** | Breadth of personal-analytics fields (workflows, releases, dependents) as future signals | SQLite store + full GitHub API surface — ADR-001 chose JSONL and read-only git on purpose |
| **OpenMetadata** ([repo](https://github.com/open-metadata/OpenMetadata)) | Enterprise metadata **knowledge graph**, lineage, classifications | The idea of an explicit, queryable classification taxonomy | Platform-scale heaviness — wrong tier for a local catalog |
| **scc** / **tokei** | Fast LOC + language stats per file pattern | Cheap per-extension counting we can mirror from `ls-tree` | External binaries |

**Synthesis.** The novel, minimal position is: **enry/Linguist-style language detection + a SOMEF-style metadata taxonomy, but resolved by a deterministic signature classifier (table + jq) over the bare-clone tree — no ML, no SQLite, no extra runtime.** Evidence-bearing labels are our differentiator over SOMEF's opaque ML output.

## Information taxonomy (what is derivable, and from where)

| Source (already fetched) | Yields |
|--------------------------|--------|
| `git ls-remote` (sentry) | HEAD SHA, reachability, default branch |
| Bare clone **tree** (`ls-tree -r`) | File inventory → key files, **extension histogram (language)**, **signature files (archetype)** |
| Manifest **contents** (`git show HEAD:<file>`) | Dependencies → archetype facets; license id; scripts/entrypoints |
| README (existing extraction) | Goal/Objectives/Flows/Requirements (verbatim) |
| Commit stream (existing) | commit count, last-active, first-commit, distinct authors (bus factor) |
| `gh` (optional, already a bridge) | description, topics, `isArchived`/`isFork`/`isTemplate`, license, stars |

Everything except the last row is **offline and deterministic**. The `gh` row is *optional enrichment*, kept in the existing bridge.

## Classification design

### A. Primary language (enry-style, jq-only)

Build an extension → language map (a trimmed Linguist table) and a histogram over `ls-tree -r --name-only`, after dropping a small vendored/generated/binary skip list (`node_modules/`, `dist/`, `vendor/`, `*.min.js`, `*.lock`, `*.svg`, …). Emit `primary_language` (top non-skipped) and `language_breakdown` (top N with file counts). No full clone, no Linguist runtime.

### B. Archetype facets (signature classifier)

A **signal table** maps a file path or a manifest dependency to one or more facets. Each match is recorded as **evidence** so the label is verifiable.

| Facet (boolean) | Representative signals (file or dependency) |
|-----------------|---------------------------------------------|
| `is_ai_related` | deps `openai`, `anthropic`, `@anthropic-ai/*`, `langchain`, `llama-index`, `transformers`, `@huggingface/*`, `ollama`, `vllm`, `cohere`, `mistralai`, `google-generativeai`, `semantic-kernel`; files `*.prompt`, `llms.txt`, `AGENTS.md`, `CLAUDE.md`, `.cursor/` |
| `is_mcp_server` | dep `@modelcontextprotocol/sdk`, `modelcontextprotocol`, `mcp`; files `mcp.json`, `server.json` with `mcpServers`; `*-mcp` naming |
| `is_web_app` | deps `next`, `react`, `vue`, `svelte`, `@angular/core`, `astro`, `remix`, `nuxt`; `vite` + `index.html`; Django/Rails/Flask/FastAPI app layout |
| `is_desktop_app` | deps `electron`, `@tauri-apps/*`, `wails`; `*.csproj` WPF/WinForms; Qt/GTK manifests |
| `is_mobile_app` | `react-native`, `expo`; `pubspec.yaml` (Flutter); `*.xcodeproj`; `android/` + `build.gradle` |
| `is_cli` / `is_console_app` | `package.json` `bin`; `[project.scripts]` / setuptools `console_scripts`; `cobra`/`urfave/cli` (Go); `clap`/`structopt` (Rust); shebang scripts in `bin/`,`scripts/` |
| `is_library` | publishable manifest with no app entrypoint; `main`/`exports`/`[lib]`; `py_modules`/`packages` |
| `is_service_api` | `Dockerfile` + server framework; `openapi.{yaml,json}`; `*.proto` |
| `is_data_ml` | `pandas`,`numpy`,`scikit-learn`,`torch`,`tensorflow`,`jax`; `*.ipynb` notebooks |
| `is_infra_iac` | `*.tf`, `helm/Chart.yaml`, k8s manifests, `ansible/`, Pulumi |
| `is_containerized` | `Dockerfile`, `docker-compose.y*ml`, `*.containerfile` |
| `is_monorepo` | `pnpm-workspace.yaml`, `turbo.json`, `nx.json`, `lerna.json`, `[workspace]` in Cargo, multiple manifests under `packages/`/`apps/` |
| `is_notebook_project` | majority `*.ipynb` |
| `is_bootstrapping_ai_app` | AI signals **+** scaffold markers (`create-*` template residue, `TODO`/placeholder-heavy, very low commit count, `AGENTS.md`/`CLAUDE.md` present) → an AI app in early bootstrap |

`is_bootstrapping_ai_app` is a *composite* rule (AI-related ∧ immaturity signals), demonstrating that facets can compose without an LLM.

### C. Primary archetype (one label for the minimal UI)

Booleans can overlap (an MCP server is also AI-related and a CLI). For the single report column, resolve one `primary_archetype` via a fixed precedence, e.g.:

```
mcp_server > desktop_app > mobile_app > web_app > service_api >
data_ml > cli > infra_iac > library > unknown
```

(Precedence is data, not code — tunable and testable.) `is_ai_related` is surfaced as an orthogonal **badge**, since "AI-related" cross-cuts every archetype.

### D. Health / activity signals (booleans + counts)

`has_readme`, `has_license` (+ best-effort SPDX id), `has_tests` (`test/`,`tests/`,`*_test.go`,`*.spec.*`), `has_ci` (`.github/workflows`), `has_docs` (`docs/`, `mkdocs.yml`), `has_dockerfile`; plus already-available `commit_count`, `last_active`, `first_commit`, and **`distinct_authors`** (a bus-factor proxy from the commit stream). Optional `gh` enrichment adds `topics`, `is_archived`, `is_fork`, `stars`.

### E. Evidence model (the differentiator)

Every facet carries the signals that set it, so any label can be audited — consistent with the OSINT "verifiable evidence" principle and unlike SOMEF's ML output:

```jsonc
"classification": {
  "schema": "1.3.0",
  "primary_archetype": "mcp_server",
  "facets": {
    "is_ai_related": true,
    "is_mcp_server": true,
    "is_cli": true,
    "is_web_app": false
    // … remaining booleans default false
  },
  "evidence": [
    { "facet": "is_mcp_server",  "kind": "dependency", "source_file": "package.json", "signal": "@modelcontextprotocol/sdk" },
    { "facet": "is_ai_related",  "kind": "dependency", "source_file": "package.json", "signal": "@anthropic-ai/sdk" },
    { "facet": "is_cli",         "kind": "manifest",   "source_file": "package.json", "signal": "bin.github-catalog-mcp" }
  ],
  "primary_language": "TypeScript",
  "language_breakdown": [ { "language": "TypeScript", "files": 38 }, { "language": "JSON", "files": 6 } ],
  "health": { "has_readme": true, "has_license": true, "license_spdx": "MIT",
              "has_tests": true, "has_ci": true, "has_dockerfile": false,
              "distinct_authors": 2 },
  "classifier_version": "1.0.0"
}
```

`classifier_version` lets you re-run classification when rules improve, while the deterministic inputs stay pinned. On a sentry **skip**, the prior `classification` is copied verbatim (same as other catalog fields) unless `classifier_version` changed.

---

## Where it runs (near-zero added cost)

Classification executes **inside `github-catalog-datafetcher.sh`**, immediately after the existing bare clone, reusing the already-fetched `ls-tree` output and `git show` for manifests. It adds a few `git show`/`jq` calls per repo — no extra network, no extra clone. The result is appended to the `repo_snapshot` as `classification`. The sentry skip path means unchanged repos pay nothing.

---

## Schema & interface impact (kept minimal)

- **Schema → 1.3.0**, additive: optional `classification` object on `repo_snapshot`. Old records (no `classification`) remain valid; consumers treat it as "unclassified".
- **CLI:** **no new commands.** Classification is produced by `sync`. (Optional later: `sync --no-classify` escape hatch.)
- **Report:** one new compact **`Type`** column in the Repository Summary (`primary_archetype` + an `AI`/`MCP` badge), and an **Archetype rollup** line in Catalog Overview (`12 web · 4 cli · 3 mcp · 2 library · 1 ai-bootstrap`). A `report --filter <facet>` is deferred to keep the surface minimal until demand is shown.
- **JSON/MCP consumers** (ADR-003 P0.3, ADR-005) get the full `classification` object for free.

## Options considered

| Option | Verdict |
|--------|---------|
| Ask `gh`/GitHub for archetype | **Impossible** — only `language` + manual `topics` exist. |
| LLM/agent classifier in core | **Rejected** — non-deterministic, heavy deps, unverifiable; violates ADR-001 ethos. (Allowed only as optional enrichment behind ADR-005.) |
| Vendor Linguist/enry binary | **Rejected** — new runtime; we only need a small extension histogram, doable in jq. |
| **Signature classifier (table + jq) over the bare clone** | **Chosen** — deterministic, evidence-bearing, zero new dependency, near-zero added cost. |

## Phased plan

1. **Language histogram** (`primary_language`, `language_breakdown`) from the existing tree + skip list. Acceptance: fixture repos classify to the expected top language.
2. **Signature classifier core** — the facet table as data; emit `facets` + `evidence` from tree + manifests. Acceptance: one fixture per archetype (web/cli/mcp/desktop/data_ml/library) asserts the right booleans **and** evidence.
3. **Composite + primary archetype + AI badge** — precedence resolution; `is_bootstrapping_ai_app`. Acceptance: overlapping fixture (MCP+CLI+AI) resolves to `mcp_server` with `is_ai_related` badge.
4. **Health signals** (`has_*`, `distinct_authors`, SPDX guess). Acceptance: fixtures.
5. **Report `Type` column + Overview rollup**; schema bump to 1.3.0; skip-path copies `classification`. Acceptance: smoke + report tests.
6. *(Optional, later)* `gh` enrichment (topics/archived/stars); `report --filter`.

## Consequences

**Positive.** Answers "what kind of project is each repo?" deterministically and verifiably; no new dependency; the report stays minimal (one column + one rollup line); the rich object empowers JSON/MCP consumers; evidence makes every label auditable.

**Negative / watch.** The facet table is **maintenance** — new frameworks/dependencies need new signals (mitigated by keeping it as data with `classifier_version`). Naming/path heuristics have false positives (mitigated by evidence + precedence + per-archetype fixtures). Manifest reads add a small per-repo cost (mitigated by the sentry skip path). `primary_archetype` precedence is a judgment call and must be unit-tested as data.

**Out of scope.** Fuzzy/subjective labels and natural-language capability summaries — offered only as optional, clearly-marked LLM enrichment via [ADR-005](ADR-005.md), never inside the deterministic `classification`.
