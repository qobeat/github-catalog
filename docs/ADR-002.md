# ADR-002: github-catalog — User Experience (UX) and Agent Experience (AX) 
**Status:** Depricated  
**Date:** 2026-06-17  
**Environment:** WSL2 · Linux 6.6.114.1-microsoft-standard-WSL2 · x86\_64 · Bash 5.0+ · jq 1.7  · gh 2.45.0
**Replaces:** Previous architecture

While the pure-Bash constraints and JSONL logic from ADR-001 are successfully implemented, the **User Experience (UX) and Agent Experience (AX) are currently fragmented**.

There are multiple entrypoints (`github-gh.sh`, `orchestrator.sh`, `qobeat-repos.sh`, `test.sh`, `lint.sh`) scattered across directories with verbose flags. A "best-in-class, minimalistic interface" requires a unified control plane.

### The Refactoring Strategy (Breaking Changes Applied)

1. **The Unified CLI (`github-catalog`)**: I am introducing a single, top-level executable at the root of the project. This CLI intercepts simple commands (`sync`, `report`, `test`, `lint`) and auto-wires the verbose flags and internal paths for the underlying scripts.
2. **Path Auto-Resolution**: The report generator will now automatically locate `data/<owner>/` and output to `reports/<owner>/latest.md` just by passing the owner name.
3. **Manifest Consolidation**: Having three `MANIFEST.md` files confuses LLM context windows. I have consolidated all agent instructions into a single, authoritative `MANIFEST.md` at the root.
4. **Deprecation**: You can delete `scripts/qobeat-repos.sh`. The new CLI makes it obsolete by offering positional arguments (e.g., `./github-catalog sync qobeat "ados-*"`).

Here are the updated files to establish this premium interface.

### Post-Generation Cleanup Instructions

To finalize this "minimalistic" refactor, please run the following commands in your terminal to remove the deprecated/redundant files:

```bash
chmod +x github-catalog
rm scripts/qobeat-repos.sh
rm scripts/MANIFEST.md
rm tests/MANIFEST.md
*Note: The unified `MANIFEST.md` at the root now handles all agent instructions beautifully and concisely.*

```
